## To do — first Linux session (Lima VM or any Ubuntu 22.04 box)

1. ⬜ `limactl start --name=meridian test/vm/meridian.yaml` (or provision a 5.15 box with `test/vm/provision.sh`)
2. ⬜ `make doctor` — all kernel config checks green
3. ⬜ `go mod tidy` — resolve `go.sum` (network required; verify cilium/ebpf v0.17.x + x/sys pins)
4. ⬜ `make vmlinux` — generate and **commit** `bpf/include/vmlinux.h`
5. ⬜ `make ebpf` — first bpf2go run; **commit** `bpf/counter_bpfel.go` + `bpf/counter_bpfel.o`.
   Verify the generated identifiers match the Go code's assumptions:
   `bpf.CounterObjects{MeridianCounter, FlowEvents, MetricsMap, SchemaSentinelMap}` and
   `bpf.CounterFlowEvent{TimestampNs, SrcIp, DstIp, SrcPort, DstPort, Proto, Verdict, SrcIdentity, DstIdentity, Bytes, LatencyNs, L7StatusCode}`
   — adjust `consumer_linux.go` / `loader_linux.go` / tests if bpf2go's casing differs.
6. ⬜ `go build ./...` and `make build` — everything compiles on Linux
7. ⬜ `make test-unit` — T1 green (also run on the macOS host: portable packages must pass there too)
8. ⬜ `make test-bpf` — **gate P0.1/P1-prep:** program loads verifier-clean; synthetic packet returns `TC_ACT_OK` and increments the counter
9. ⬜ `make test-integration` — **gates P0.2 + O-1:** ping traffic counted in the PERCPU map AND decoded by the real ring consumer with correct addresses
10. ⬜ `make verify-gen` — run twice; second run must produce zero diff (determinism gate)
11. ⬜ Push and confirm all three CI jobs green (CI cannot pass before steps 4–5 are committed)
